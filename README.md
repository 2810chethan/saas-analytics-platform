# AetherSaaS BI - Analytics & Churn Prediction Platform

A highly polished, interactive business intelligence workspace designed for subscription-based SaaS platforms. It includes a multi-variable synthetic cohort generator, an analytics metric processor with churn risk classification modeling, and a gorgeous glassmorphic dark-theme browser dashboard featuring standard financial charts, 12-month cohort retention heatmaps, customer search engines, and a real-time "What-If" business strategy projection simulator.

## 🚀 Quick Start Guide

### 1. Generate & Process Platform Data
Make sure you have Python installed on your system. Open your terminal in this directory and execute the following two commands:

```bash
# Step 1: Generate 24 months of raw cohort customer records & activity logs
python data_generator.py

# Step 2: Compute core SaaS KPIs, cohort heatmaps, and score churn risk
python analyzer.py
```

- `data_generator.py` will generate ~2,000 customers and write them to `data/users.csv` and `data/activity_logs.csv`.
- `analyzer.py` will process those raw tables, run mathematical metrics calculations, run a custom logistic risk scoring classification, and export the structured dataset to `dashboard_data.json`.

### 2. View the Premium Web Dashboard
Since the front-end workspace reads compiled database records directly from `dashboard_data.json` via fetch requests, it should be opened through a local web server to comply with browser CORS policies.

**Option A: Instant Python Server (Recommended)**
Run this single command in your terminal:
```bash
python -m http.server 8000
```
Then, open your web browser and navigate to:
👉 **[http://localhost:8000/login.html](http://localhost:8000/login.html)**

**Option B: Visual Studio Code "Live Server" Extension**
If you use VS Code, right-click `login.html` and click **"Open with Live Server"**.

## 📈 Formulated SaaS Metrics Glossary

The analytics engine (`analyzer.py`) calculates traditional SaaS metrics as defined below:

1. **Monthly Recurring Revenue (MRR):**
   $$\text{MRR} = \sum (\text{Active Subscribers' Plan Prices in Month } m)$$
   Tracks core financial viability month-over-month.

2. **Annual Recurring Revenue (ARR):**
   $$\text{ARR} = \text{MRR} \times 12$$
   The forward-looking annual run rate of the business.

3. **Average Revenue Per User (ARPU):**
   $$\text{ARPU} = \frac{\text{MRR}}{\text{Total Active Subscribers}}$$

4. **Monthly Churn Rate:**
   $$\text{Churn Rate} = \frac{\text{Subscriptions Cancelled in Month } m}{\text{Active Subscribers at Start of Month } m}$$
   Measures structural customer attrition. Capped at a minimum of 1.5% to avoid division outliers in early phases.

5. **Customer Lifetime Value (LTV):**
   $$\text{LTV} = \frac{\text{ARPU} \times \text{Gross Margin (85\%)}}{\text{Effective Monthly Churn Rate}}$$
   Projects compound future earnings of a blended subscriber profile.

6. **Customer Acquisition Cost (CAC):**
   Calculated as the blended marketing acquisition expense required to acquire new subscribers signed up in that month cohort.

7. **Net Revenue Retention (NRR):**
   $$\text{NRR} = \frac{\text{Revenue Retained in Month } m \text{ from Month } m-1 \text{ Cohort}}{\text{Total Revenue generated by Cohort in Month } m-1} \times 100$$
   Tracks internal accounts growth (upgrades vs downgrades and cancellations). NRR > 100% shows natural growth.

8. **Gross Revenue Retention (GRR):**
   Identical to NRR, but caps individual account expansions at 100% (exclusing plan upgrades), measuring absolute base revenue retention stability.

## 🧠 Churn Risk Classification Model

Instead of generic placeholders, the platform integrates a mathematical **Logistic Sigmoid Risk Scorer** inside `analyzer.py`. For every active subscriber at the current calendar month, the engine reviews:
- **Login Inactivity Recency:** Decreasing login frequency spikes risk (+1.8 log odds).
- **Product Module Utilization:** Low feature scores signify stagnation (+1.4 log odds).
- **Service Friction:** Escalated support tickets indicate immediate frustration (+1.2 log odds).
- **Financial Friction:** Payment processing failures flag structural churn (+2.2 log odds).
- **Lifecycle Stability:** Longer customer tenures exhibit lower baseline churn risk (-0.9 log odds).

The combined log odds ($z$) are mapped to a final probability curve:
$$\text{Risk Probability} = \frac{1}{1 + e^{-z}}$$

Customers with **Risk Probability > 70%** are highlighted as **High Risk**, enabling management to perform proactive outreach campaigns via direct contact buttons embedded in the list.

## 🎛️ Strategic "What-If" Planning Tool

The **What-If Strategy Planner** tab lets managers slide business levers to predict ARR scaling over the next 12 months:
- **Price Optimization:** Simulates adjustments (+/-) to plans, directly altering ARPU.
- **Churn Mitigation:** Simulates active efforts to improve retention, reducing compound decay.
- **Acquisition Efficiency:** Simulates lowering CAC, allowing more signups for equivalent marketing budgets.
